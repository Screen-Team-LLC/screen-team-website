# Security Policy

## Supported Version

Only the current `main` branch is supported.

## Reporting a Vulnerability

Do not open a public issue for security problems.

If you discover a vulnerability involving this site, its forms, deployment, DNS, or analytics integrations:

1. Report it privately through GitHub security reporting when available.
2. If private GitHub reporting is not practical, contact the maintainers directly before public disclosure.
3. Include reproduction steps, affected URLs, and any relevant screenshots or request details.

## Scope

Security-sensitive areas for this repository include:

- GitHub Pages deployment and custom-domain configuration
- Contact-form handling and third-party form integrations
- Analytics or verification snippets
- Search/indexing automation
- Any secrets, tokens, or workflow credentials

## Response Expectations

Reasonable good-faith reports will be reviewed as quickly as possible. Please avoid public disclosure until maintainers have had time to assess and remediate the issue.
